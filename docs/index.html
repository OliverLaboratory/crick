How it works

Proof of work secures a blockchain by making blocks expensive to produce. crick keeps that security intact and adds one rule that makes solving real scientific problems the rational way to earn a block — so the same effort that protects the chain also produces useful results.

01 · TWO WAYS TO MINE

Solve, or hash

Every block can be mined classically at the base difficulty d_b, or — if it carries a strictly better solution to the network's puzzle — at a reduced difficulty d_r < d_b. Improving the solution is cheaper than out-hashing the network.

02 · DIPS v2 RETARGETING

Independent difficulties

Each difficulty retargets from its own block type: d_b holds classical production steady, d_r holds solution production steady. When a full window passes with no solution found, d_r drops 4× — the harder the problem gets, the deeper the discount for solving it.

03 · EPOCHS

One instance at a time, then rotate

The chain optimizes one problem instance per epoch, then rotates when it saturates. The next instance comes from the rotating block's hash — unpredictable (no precomputing the corpus) and unchooseable (no cherry-picking the easiest one). d_b stays global; d_r resets each epoch so a fresh instance can't be farmed for cheap discounted blocks.

04 · USEFUL OUTPUT

Science on the ledger

The chain becomes a public, append-only record of ever-improving biological solutions — lowest-energy ligand poses, shared structural motifs — drawn from billions × billions of structures, so vast that precomputing them is the science.

Problems you can solve

A network is seeded with one problem type at genesis and then works through a corpus of its instances, one per epoch (see above). Every type shares the same protocol and differs only in its (cheap, exact, integer) verifier — so "is this a real improvement?" is always an unambiguous comparison. Choose one with crick init --problem.

DOCKING · default

Protein–ligand docking

Find a ligand pose with lower energy than the best so far, scored by a fixed integer energy function on a 3D lattice. Verifying a pose is one integer sum; finding a good one means searching rotations × translations. Instances are drawn from ESM Atlas × a ligand library (Enamine).

crick init --problem docking
MCS-PROTEIN

Maximum common subgraph

The largest shared substructure between two protein contact graphs — exactly max-clique on their modular product. A solution is a residue↔residue mapping; the chain accumulates a public library of shared structural motifs.

crick init --problem mcs-protein
MAX-CLIQUE

Maximum clique

The reference / benchmark problem: largest clique in a pseudorandom graph derived from the network seed. Trivial exact verification, short improvement ladder — useful for testing the protocol itself.

crick init --problem max-clique

Propose a new problem — forking

Rotating to the next instance of the same problem type happens automatically each epoch. Introducing a whole new kind of problem — a new verifier — is a deliberate fork the community opts into. Two steps:

# 1. implement the verifier. it must be cheap, exact, and deterministic,
#    with an integer score where higher is strictly better.
@register_problem
class FoldingProblem(Problem):
    type = "folding"
    def verify(self, solution): ...   # poly-time, deterministic
    def score(self, solution): ...    # integer; higher = better
    def improve(self, best): ...      # miner-side search

# 2. fork: agree off-chain, then launch a chain seeded with it
$ crick init --problem folding

The papers also sketch an on-chain path: a stake-gated submission transaction where the proposer burns tokens, so flooding the network with junk or pre-solved problems is never profitable — keeping problem selection decentralized without a central authority.

Install

crick is a single Python package with one small dependency. Python 3.9+ required.

# from source
$ git clone https://github.com/OliverLaboratory/crick.git && cd crick
$ pip install .

# or, isolated install with pipx
$ pipx install git+https://github.com/OliverLaboratory/crick.git

That's it — the crick command is now on your PATH.

Quickstart

Create a wallet and genesis block, then start mining — solo or against the network.

$ crick init                  # wallet + genesis block in ~/.crick
$ crick mine                  # mine solo: solver + hasher in one loop
$ crick node --mine \
        --peer http://seed.example:9911   # join a network and mine
$ crick status                # chain height, difficulties, balance
$ crick solution              # the best on-chain solution so far
$ crick send --to crk1… --amount 5   # spend your rewards
CommandWhat it does
crick initCreate a wallet and the genesis block (stores the puzzle instance)
crick mineMine solo — alternates puzzle solving with nonce search
crick nodeRun a full node: HTTP API, peer gossip, optional mining
crick statusHeight, total work, d_b, d_r, best clique, balance
crick solutionShow and verify the best solution recorded on-chain
crick sendSign and broadcast a transaction via a running node
